## Codebase Patterns

- Solution file is at repo root `Skoruba.Duende.IdentityServer.Admin.sln`, NOT at `src/`. PRD references `src/Skoruba...sln` which does not exist.
- Test projects are in `tests/` with actual names: `Skoruba.Duende.IdentityServer.Admin.UnitTests`, `Skoruba.Duende.IdentityServer.Admin.Api.UnitTests`, `Skoruba.Duende.IdentityServer.Admin.Api.IntegrationTests`, `Skoruba.Duende.IdentityServer.STS.Identity.IntegrationTests`.
- Integration tests use in-memory EF databases via `RegisterDbContextsStaging` — no external SQL Server required.
- dotnet build triggers `npm install` for the SPA project automatically via `Directory.Build.targets`. `npm ci` standalone cannot reach the registry from sandbox (network blocked). Use `npm run build:spa` from `node_modules` that dotnet build populated.
- SPA build script (`scripts/build-and-copy.js`) must be run via `npm run build:spa`, not `node scripts/build-and-copy.js` directly — npm run adds `node_modules/.bin/` to PATH (needed for `vite`).
- Working tree shows mass CRLF noise on checkout in this sandbox (no `.gitattributes`). Always run `git checkout -- .` at session start to clean.
