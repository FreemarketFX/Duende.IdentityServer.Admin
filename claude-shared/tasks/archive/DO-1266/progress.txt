## Codebase Patterns
- PowerShell scripts use `$PSScriptRoot` for script directory (equivalent to Python's `Path(__file__).parent`)
- Use `[Console]::In.ReadToEnd()` for stdin reading in PowerShell hooks
- `ConvertFrom-Json` returns PSCustomObject, not hashtable — use `$obj.PSObject.Properties[$name]` for safe property access
- `ConvertFrom-Yaml` (powershell-yaml) returns ordered dictionaries — use `.ContainsKey()` for hashtable checks
- Both PSCustomObject and IDictionary must be handled for property access since different input sources produce different types
- The `trap { exit 0 }` block at script level provides fail-open behavior for any unhandled errors
- `$MyInvocation.InvocationName -ne '.'` distinguishes direct execution from dot-sourcing (needed for test script import)
